# Project

Describe the project here.

## Goals

- Keep the project maintainable.
- Keep public APIs clear and stable.
- Prefer simple, testable implementation.

## Non-goals

- Do not add unnecessary abstractions.
- Do not introduce new runtime dependencies without a clear reason.
- Do not change public APIs unless explicitly requested.

## Architecture boundaries

- Keep core logic independent from framework-specific code.
- Keep data processing separate from rendering or UI logic.
- Avoid duplicated large data structures.
- Keep modules small and focused.
